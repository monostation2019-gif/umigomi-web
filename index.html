# 海ゴミ探究ノート（HTML/JS版）v1.0 ターミナルもインストール作業も不要です。ファイルを編集して、Vercelにアップロードするだけで動きます。 ## 構成 ``` umigomi_web/ ├── index.html ← アプリ本体（これ1つに画面・処理すべてが入っています） ├── manifest.json ← PWA設定 ├── sw.js ← オフライン対応（そのままでOK、触らなくて大丈夫） ├── favicon.png └── icons/ ← アプリアイコン・初期プロフィール画像 ``` ## v1.0で実装済みの機能 - ログイン（Googleログイン／ゲスト（匿名）ログイン） - 自動ログイン（一度ログインすれば次回から自動） - ホーム画面（歓迎メッセージ・記録サマリー・他タブへのショートカット・使い方） - プロフィール画面（ニックネーム・ひとこと・作成日時・更新日時の表示と編集） - プロフィール画像の変更（アルバムから選択→自動圧縮→保存。Storage不要でFirestoreのみで完結） - ボトムナビ4タブ（ホーム／地図／記録一覧／プロフィール、切り替えのみ） - 藍色ヘッダー・ボトムナビのデザイン **まだ実装していないもの**：地図・記録一覧の中身（現在は仮表示） ## セットアップ手順 ### ① Firebaseの接続情報を書き込む 1. `index.html` を **メモ帳（またはVSCode）** で開く 2. `Ctrl + F` で `REPLACE_ME` を検索する（6箇所あります） 3. Firebaseコンソール（https://console.firebase.google.com）を開き、 - 左上の**⚙️歯車アイコン → プロジェクトの設定**をクリック - 下の方の「マイアプリ」に、既にWebアプリが登録されていればそれをクリック （無ければ `` アイコンをクリックして新規登録） - 表示される `firebaseConfig = { ... }` の中身をまるごとコピー 4. `index.html` 内の該当部分を、コピーした内容で置き換えて保存する ### ② Firebaseコンソールで設定する項目 - **Authentication → Sign-in method** - 「匿名」を有効化 - 「Google」を有効化（Googleログインを使う場合） - **Firestore Database** を作成する（無ければ「データベースを作成」） - **Firestore Database → ルール** タブで、以下を貼り付けて「公開」 ``` rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{uid} { allow read, write: if request.auth != null && request.auth.uid == uid; } } } ``` > Firebase Storageは使用していないため、設定不要です。 ## 動作確認の方法 **方法A（一番簡単）：そのままダブルクリックで開く** `index.html` をダブルクリックしてブラウザで開くだけで動作確認できます。 **方法B（本番と同じ環境で確認したい場合）：Vercelにアップロード** `umigomi_web` フォルダを丸ごとVercelにドラッグ＆ドロップしてください。 ## 動かない時のチェックリスト | 症状 | 原因の可能性 | |---|---| | ずっと「準備中…」のまま | `REPLACE_ME` が残っている / Firebaseの値が間違っている | | 赤い文字でエラーが出る | エラー文をそのままコピーして質問してください | | ログインできない | Authentication の「匿名」または「Google」が無効になっている | | プロフィールが表示されない／権限エラー | Firestore Database未作成、またはルール未設定 | ## 今後の拡張予定（v1.1以降） - 地図タブの実装（GPS・ピン設置・タイル表示） - 記録一覧タブの実装（ゴミ記録の投稿・分類・一覧表示） - ホームの「これまでの記録：0件」を実際の件数に接続 ## 変更履歴 - **v1.0**：ゲスト/Googleログイン、プロフィール（画像変更含む）、ホーム画面、ボトムナビの土台が完成
