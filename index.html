# 海ゴミ探究ノート（HTML/JS版・土台） ターミナルもインストール作業も不要です。ファイルを編集して、Vercelにアップロードするだけで動きます。 ## 構成 ``` umigomi_web/ ├── index.html ← アプリ本体（これ1つに画面・処理すべてが入っています） ├── manifest.json ← PWA設定 ├── sw.js ← オフライン対応（そのままでOK、触らなくて大丈夫） ├── favicon.png └── icons/ ← アプリアイコン・初期プロフィール画像 ``` ## やることは2つだけです ### ① Firebaseの接続情報を書き込む 1. `index.html` を **メモ帳（またはVSCode）** で開く 2. `Ctrl + F` で `REPLACE_ME` を検索する（6箇所あります） 3. Firebaseコンソール（https://console.firebase.google.com）を開き、 - 左上の**⚙️歯車アイコン → プロジェクトの設定**をクリック - 下の方の「マイアプリ」に、既にWebアプリが登録されていればそれをクリック （無ければ `` アイコンをクリックして新規登録） - 表示される `firebaseConfig = { ... }` の中身をまるごとコピー 4. `index.html` 内の以下の部分を、コピーした内容で置き換えて保存する ```js const firebaseConfig = { apiKey: "REPLACE_ME", authDomain: "REPLACE_ME.firebaseapp.com", projectId: "REPLACE_ME", storageBucket: "REPLACE_ME.appspot.com", messagingSenderId: "REPLACE_ME", appId: "REPLACE_ME" }; ``` ### ② Firebaseコンソールで2つだけ設定を確認する - **Authentication** → 「Sign-in method」タブ → **「匿名」を有効化** - **Firestore Database** が作成済みであること（無ければ「データベースを作成」） ## 動作確認の方法 **方法A（一番簡単）：そのままダブルクリックで開く** `index.html` をダブルクリックしてブラウザで開くだけで動作確認できます。 （PWAのオフライン機能だけは、この方法だとオフには動きませんが、ログイン・プロフィール表示は問題なく確認できます） **方法B（本番と同じ環境で確認したい場合）：Vercelにアップロード** 今までと同じように、`umigomi_web` フォルダを丸ごとVercelにドラッグ＆ドロップしてください。 これでURLが発行され、スマホからもPWAとしてインストールして確認できます。 ## 動かない時のチェックリスト | 症状 | 原因の可能性 | |---|---| | ずっと「準備中…」のまま | `REPLACE_ME` が残っている / Firebaseの値が間違っている | | 赤い文字でエラーが出る | エラー文をそのままコピーして質問してください | | ログインできない | Authentication の「匿名」が無効になっている | | プロフィールが表示されない | Firestore Database が作成されていない | ## 今後の拡張予定 - 地図タブ・記録一覧タブは今は仮表示のみです - プロフィール画像のアップロードは「編集する」→保存の仕組みは既にあるので、 画像選択機能を追加すれば `iconUrl` を更新するだけで反映されます
